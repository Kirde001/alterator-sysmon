System Monitoring (eBPF)

This module is designed to monitor suspicious system activity. Using eBPF technology, it intercepts Linux kernel system calls, allowing you to record process execution and access to critical files.

Key Features

  • Process Analysis: Capture all program executions (including command line arguments).
  • File Control: Monitor attempts to read or write to critical system files (e.g., /etc/shadow, /etc/passwd).
  • SIEM Integration: Ability to send security events to external systems (Wazuh, ArcSight) via Syslog.

Module Settings

The following options are available at the top of the window:

  • Enable Syslog/SIEM: If checked, events will be duplicated to the system log.
  • Log Format: Select the data format (RFC 3164, JSON, or CEF).

Event Table

The table displays the latest recorded events:

  • Severity: Threat level (Critical, High, Medium, Low).
  • Event Type: process_execution or sensitive_file_access.
  • User: The real username of the user who initiated the action (even when using sudo).
  • Process: The program that performed the action.
  • Details: Detailed information (file path, parent process).

Rules Configuration

Fine-tuning of filtering is done via the configuration file:

/etc/syscall-inspector/rules.json

You can configure lists of ignored processes and file extensions to reduce "noise" in the logs.
